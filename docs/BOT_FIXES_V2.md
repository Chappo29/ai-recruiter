## 🐛 Обнаруженные проблемы

### 1. **Бот галлюцинирует (спрашивает про то, чего нет в резюме)**

**Пример:**
```
Резюме: Менеджер по продажам (опыт с CRM, B2B, рекрутинг)
Вакансия: Python Developer

Бот: "Кажется, вы указали Python в навыках. Можете рассказать о вашем 
      опыте работы с этим языком?"
```

❌ **Проблема:** Python НЕТ в резюме, бот придумал!

**Причина:** LLM видит название вакансии "Python Developer" и пытается найти соответствие, выдумывая факты.

---

### 2. **Хамский/обвинительный стиль общения**

**Примеры плохих фраз:**
- ❌ "Понятно. Вы не указали никакой мотивации для данной позиции."
- ❌ "Кажется, вы не указали мотивацию для позиции Python Developer."
- ❌ "Вы не указали мотивацию..."

**Проблема:** Звучит как обвинение, упрек, давление на кандидата.

**Правильно должно быть:**
- ✅ "Расскажите, пожалуйста, почему вас заинтересовала эта позиция?"
- ✅ "Какой у вас опыт работы с..."

---

### 3. **Плохая реакция на оскорбления**

**Пример:**
```
Кандидат: Пошел нахуй
Бот: "Понимаю ваше недовольство. Давайте вернёмся к вашему опыте и 
      мотивации по вакансии Python Developer."
```

❌ **Проблема:** "Понимаю ваше недовольство" звучит саркастически + бот продолжает диалог вместо завершения.

**Правильно:**
- ✅ Мягко завершить диалог: "Спасибо за время! Передам вашу анкету рекрутеру."

---

### 4. **Баг: вопрос + "Удачи!" одновременно**

**Пример:**
```
Бот: Кажется, вы не указали мотивацию для позиции Python Developer. 
     Почему вас интересует эта роль?
Бот: Спасибо за ваши ответы, Владислав! 🙏
     Ваша кандидатура передана рекрутеру.
     Удачи! 🍀
```

❌ **Проблема:** Задал вопрос и сразу завершил диалог - логика сломана.

---

### 5. **Нет защиты от "не резюме" в PDF**

**Проблема:** Кандидат может отправить любой PDF (договор, инструкцию, счет-фактуру), и бот примет его как резюме.

❌ **Нет валидации** что PDF содержит резюме.

### Исправление 1: Защита от галлюцинаций

**Файл:** `app/services/llm_service.py`

**Добавлено в промпт:**

```python
Анализ резюме — ТОЛЬКО ФАКТЫ:
- НИКОГДА не придумывай что есть в резюме. Если в резюме НЕТ упоминания 
  технологии/навыка — НЕ говори "вы указали".
- Если резюме СОВСЕМ не подходит по вакансии (например менеджер на вакансию 
  разработчика) — мягко заверши ("done": true) с: "Спасибо! Ваша анкета 
  передана рекрутеру."
- Ищи в резюме пробелы и недосказанности ТОЛЬКО по тому что там РЕАЛЬНО написано.
```

**Эффект:**
- ✅ Бот НЕ будет спрашивать про Python, если его нет в резюме
- ✅ Бот мягко завершит диалог при явном несоответствии (менеджер → Python Dev)

### Исправление 2: Вежливый стиль общения

**Файл:** `app/services/llm_service.py`

Стиль общения — ВЕЖЛИВЫЙ И ЭМПАТИЧНЫЙ:

### Исправление 4: Early exit (мягкое завершение)

**Файл:** `app/services/llm_service.py`

**Добавлено в промпт:**

```python
- Если кандидат грубит или оскорбляет — СРАЗУ заверши диалог ("done": true) 
  с сообщением: "Спасибо за время! Передам вашу анкету рекрутеру."
- Если кандидат отвечает коротко или уклончиво 2+ раза подряд — мягко заверши 
  ("done": true) с сообщением: "Спасибо! Передам анкету рекрутеру."
```

**Условия завершения:**

1. **Оскорбления** → завершение
2. **Резюме не подходит** (менеджер на вакансию разработчика) → завершение
3. **Кандидат уклоняется 2+ раза** ("нет", "никакие", "не знаю") → завершение
4. **Кандидат ответил на все вопросы** → завершение

**Эффект:**
- ✅ Нет бага "вопрос + Удачи одновременно"
- ✅ Диалог завершается корректно

### Исправление 5: Валидация PDF резюме

**Новый файл:** `app/utils/resume_validation.py`

**Функции:**

```python
def validate_resume_comprehensive(text: str) -> dict:
    """
    Полная валидация резюме с несколькими проверками.
    
    Проверки:
    1. Длина текста (минимум 50 символов)
    2. Ключевые слова резюме (опыт работы, навыки, образование, и т.д.)
    3. Наличие ФИО в начале документа
    4. Отсутствие подозрительных паттернов (счет-фактура, договор, инструкция)
    
    Returns:
        {
            "is_valid": bool,
            "confidence": float,
            "checks": {...},
            "reason": str,
        }
    """
```

**Ключевые слова для проверки:**
- Русские: `опыт работы`, `образование`, `навыки`, `должность`, `компания`, `проект`, `достижения`, `резюме`
- Английские: `experience`, `education`, `skills`, `position`, `company`, `projects`, `resume`, `cv`

**Подозрительные паттерны (НЕ резюме):**
- `счет-фактура`, `договор`, `соглашение`, `инструкция`, `отчет`, `презентация`
- `invoice`, `contract`, `agreement`, `instructions`, `report`, `presentation`

**Интеграция в бота:**

```python
# bot/resume_pipeline.py
from app.utils.resume_validation import validate_resume_comprehensive

validation = validate_resume_comprehensive(resume_text)
if not validation["is_valid"]:
    await bot.send_message(
        chat_id,
        "Кажется, это не резюме 🤔\n\n"
        "Пожалуйста, отправьте PDF-файл с вашим резюме.\n"
        "Как скачать с hh.ru:\n"
        "1. Зайдите на hh.ru → Моё резюме\n"
        "2. Нажмите «Скачать» → PDF\n"
        "3. Отправьте файл сюда",
    )
    return
```

**Эффект:**
- ✅ Если PDF не резюме (например инструкция) → отклонение с понятным сообщением
- ✅ Минимум 2 ключевых слова резюме + наличие ФИО → проходит валидацию

Протестировать варианты:
- "Спасибо! Передам анкету рекрутеру."
- "Спасибо за время! Рекрутер рассмотрит вашу анкету."

Добавить проверку:
