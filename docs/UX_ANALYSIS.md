# UX-анализ: Путь рекрутера в системе

## 🎯 Цель анализа
Оценить, насколько интуитивен и удобен интерфейс для рекрутеров, которые будут работать с кандидатами ежедневно.

---

## 👤 Целевая аудитория

**Рекрутер:**
- Возраст: 25-45 лет
- Опыт работы: от начинающего до опытного
- Технические навыки: базовые (умеет работать с CRM, почтой, Telegram)
- Задачи: просмотр откликов, оценка кандидатов, проведение первичного отбора
- Боли: много кандидатов, нужно быстро принимать решения, не терять хороших кандидатов

---

## 📊 Анализ пользовательского пути

### Сценарий 1: Первый заход на страницу кандидатов

#### ✅ Что хорошо:

1. **Визуальная понятность Kanban**
   - 4 колонки с понятными русскими названиями: "Новые", "AI проверил", "Собеседование", "Отклонено"
   - Цветовая кодировка (серый → индиго → зеленый → красный) логична
   - Счетчики кандидатов в заголовках

2. **Dropdown вакансии сверху**
   - Всегда виден, не пропадает
   - Понятно, для какой вакансии показаны кандидаты

3. **Компактные карточки**
   - Аватар + имя = быстрая идентификация
   - Score badge яркий и заметный
   - Время отклика ("2 дн. назад") помогает приоритизировать

#### ⚠️ Потенциальные проблемы:

1. **Нет объяснения, что можно перетаскивать**
   - Рекрутер может не догадаться, что карточки можно тащить
   - **Решение:** Добавить подсказку при первом заходе или иконку "↔" на карточках

2. **Непонятно, чем "Новые" отличаются от "AI проверил"**
   - Оба имеют статус "pending"
   - **Решение:** Добавить в описание колонки: "Новые: ожидают AI-анализа", "AI проверил: анализ завершен, есть оценка"

3. **Нет явной подсказки про клик для деталей**
   - Hover эффект есть, но не все пользователи его заметят
   - **Решение:** Добавить иконку "👁" или "🔍" при наведении

---

### Сценарий 2: Просмотр деталей кандидата

#### ✅ Что хорошо:

1. **Клик на карточку → модалка**
   - Интуитивное действие
   - Модалка содержит ВСЮ информацию (score, strengths, weaknesses, резюме, история)
   - Можно закрыть через X, Escape, клик вне окна

2. **Вся информация в одном месте**
   - Не нужно переходить на другие страницы
   - Резюме можно открыть/скачать прямо из модалки
   - История откликов сразу видна

3. **Кнопки действий понятны**
   - "Открыть в Telegram" → связаться с кандидатом
   - "Передан дальше" → на собеседование
   - "Отклонить" → отказ

#### ⚠️ Потенциальные проблемы:

1. **Слишком много информации в модалке**
   - При большом резюме нужно много скроллить
   - **Решение:** Уже решено - скроллируемая модалка с sticky кнопкой закрытия

2. **Нет быстрых действий прямо на карточке**
   - Приходится открывать модалку для каждого действия
   - **Решение (Phase 2):** Добавить иконки действий на hover (✓, ✗, ✉)

3. **Иконка робота 🤖 может быть непонятна**
   - Что это значит?
   - **Решение:** Добавлен tooltip "Подозрение на AI-резюме" ✅ (уже есть)

---

### Сценарий 3: Перетаскивание кандидата в другой этап

#### ✅ Что хорошо:

1. **Drag & Drop интуитивен**
   - Просто берешь и тащишь
   - Курсор меняется на "grab" → "grabbing"
   - Карточка следует за мышью с небольшим поворотом (3deg)

2. **Визуальная обратная связь**
   - Колонка подсвечивается пунктирной рамкой цвета стадии
   - Opacity карточки уменьшается до 0.5 при перетаскивании
   - Текст "Отпустите здесь" в пустой колонке

3. **Optimistic update**
   - Карточка мгновенно перемещается
   - Если ошибка на сервере → откатывается назад
   - Toast-уведомление о результате

#### ⚠️ Потенциальные проблемы:

1. **На мобилке drag & drop не работает**
   - Touch события отличаются от мышиных
   - **Решение (Phase 2):** Добавить сенсор для touch или кнопки действий на карточках

2. **Нет подтверждения при перетаскивании в "Отклонено"**
   - Можно случайно отклонить кандидата
   - **Решение:** Сейчас есть confirm при нажатии кнопки "Отклонить", но при drag нет
   - **Рекомендация:** Добавить подтверждение при drop в "Отклонено"

3. **Непонятно, что drag начинается после 8px движения**
   - Если не сдвинуть на 8px, будет клик вместо drag
   - **Решение:** Это норма, так работают все drag & drop системы

---

### Сценарий 4: Работа с несколькими вакансиями

#### ✅ Что хорошо:

1. **Dropdown вакансии всегда доступен**
   - Можно быстро переключаться между вакансиями
   - Не теряется при переключении (баг исправлен ✅)

2. **Каждая вакансия - отдельный Kanban**
   - Не путаются кандидаты разных вакансий
   - Счетчики показывают, сколько кандидатов в каждой стадии

#### ⚠️ Потенциальные проблемы:

1. **Нет overview всех вакансий**
   - Невозможно увидеть общую картину по всем вакансиям
   - **Решение (Phase 2):** Добавить Dashboard с общей статистикой

2. **Нет поиска по кандидатам**
   - Если кандидатов много, сложно найти конкретного
   - **Решение (Phase 2):** Добавить поле поиска над Kanban

---

### Сценарий 5: Повторные отклики

#### ✅ Что хорошо:

1. **Бейдж с номером попытки**
   - Видно сразу: [2], [3] и т.д.
   - Рекрутер понимает, что кандидат откликался раньше

2. **История откликов в модалке**
   - Можно посмотреть предыдущие результаты
   - Видно, на какие вакансии откликался

#### ⚠️ Потенциальные проблемы:

1. **Нет контекста, почему он откликнулся повторно**
   - Может, доработал резюме?
   - **Решение (будущее):** Добавить diff между версиями резюме

---

### Сценарий 6: AI-анализ в процессе

#### ✅ Что хорошо:

1. **Индикатор "ИИ анализирует N резюме..."**
   - С пульсирующей точкой
   - Понятно, что система работает

2. **Автообновление каждые 5 секунд**
   - Не нужно обновлять страницу
   - Карточки автоматически перемещаются из "Новые" в "AI проверил"

3. **Текст "ИИ анализирует..." на карточке**
   - Понятно, что нужно подождать

#### ⚠️ Потенциальные проблемы:

1. **Нет прогресс-бара**
   - Непонятно, сколько осталось ждать
   - **Решение:** AI-анализ быстрый (< 3 минут), прогресс-бар не критичен

## 🎨 Визуальный дизайн

### ✅ Сильные стороны:

1. **Консистентная цветовая схема**
   - Primary цвет (#4F46E5) используется везде: кнопки, бейджи, линки
   - Score badges с цветовой индикацией (зеленый >80, желтый 60-80, серый <60)

2. **Читаемость**
   - Шрифт Inter, размеры 12-16px
   - Хороший контраст текста на фоне
   - Иконки Lucide-react понятны

3. **Spacing и отступы**
   - Не перегружено, "дышащий" дизайн
   - Карточки имеют достаточно padding

### ⚠️ Области для улучшения:

1. **Колонка "Отклонено" может быть менее заметной**
   - Она всегда показана, но обычно не важна
   - **Решение (опционально):** Сделать её уже (280px) или затемнить

2. **Нет визуальной иерархии в модалке**
   - Всё одинаковой важности
   - **Решение:** Выделить AI Verdict ярче (уже есть VerdictBadge)

## 🚀 Критические проблемы и решения

### Проблема 1: Нет онбординга для новых пользователей
**Решение:**
- Добавить tooltip подсказку при первом заходе: "Перетащите карточку в другую колонку, чтобы изменить статус"
- Добавить "?" иконку с подсказками в правом верхнем углу

### Проблема 2: Drag & Drop может быть случайным
**Решение:**
- Добавить confirm диалог при drop в "Отклонено" ✅ (нужно добавить)
- Увеличить activationConstraint до 10-12px (сейчас 8px)

### Проблема 3: Мобильная версия не продумана
**Решение:**
- Переключить на tabs для мобилки (< 768px)
- Добавить кнопки действий на карточках вместо drag & drop

### Проблема 4: Нет обратной связи при ошибках
**Решение:**
- Toast уведомления уже есть ✅
- Добавить retry кнопку в toast при ошибке

### Важные (улучшат опыт):

   - Фильтрация по имени в реальном времени
