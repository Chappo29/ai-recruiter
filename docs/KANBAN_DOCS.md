# Kanban-доска для кандидатов - Документация

## 🎉 Что изменилось

Страница "Кандидаты" теперь отображается в виде **Kanban-доски** с колонками для разных этапов воронки найма.

---

## 📊 Структура колонок

### 1. **Новые** (New)
- Кандидаты, которые только откликнулись
- ИИ еще анализирует резюме или анализ в процессе
- **Условие**: `status === 'pending'` И (`score === null` ИЛИ AI анализ в процессе)

### 2. **AI проверил** (AI Screened)
- Кандидаты, прошедшие автоматический AI-скрининг
- Есть оценка (score) и AI-анализ завершен
- **Условие**: `status === 'pending'` И `score !== null` И AI анализ завершен

### 3. **Собеседование** (Interview)
- Кандидаты, переданные на следующий этап HR или менеджеру
- **Условие**: `status === 'forwarded'`

### 4. **Отклонено** (Rejected)
- Кандидаты, которые не подошли
- **Условие**: `status === 'rejected'`
- **По умолчанию скрыто** - можно показать через чекбокс "Показать отклоненных"

---

## 🎯 Основные возможности

### ✅ Сохранены все фишки из старой версии:

1. **Автообновление при AI-анализе**
   - Каждые 5 секунд обновляется список, если есть кандидаты в процессе анализа
   - Индикатор "ИИ анализирует N резюме..." в правом верхнем углу

2. **Фильтр повторных кандидатов**
   - Чекбокс "Повторные" - показывать/скрывать кандидатов с `screening_index > 1`
   - Бейдж "2", "3" и т.д. на карточках повторных кандидатов

3. **Полная детализация в модальном окне**
   - Клик на карточку открывает модалку с полной информацией
   - Все секции: AI Score, Сильные/Слабые стороны, AI-резюме детекция, История откликов, Лог диалога
   - Кнопки действий: "Открыть в Telegram", "Передан дальше", "Отклонить"

4. **Повторный анализ**
   - Если AI анализ не получен (ошибка/таймаут), показывается кнопка "Повторить анализ"

5. **Все остальное**
   - Empty State при отсутствии кандидатов
   - Селектор вакансии (если не указан в URL)
   - Deep-linking на конкретную вакансию через `?vacancy_id=...`
   - Скелетоны при загрузке

## 🎨 Компоненты

### Новые компоненты:

#### `KanbanBoard.jsx`
Основной контейнер Kanban-доски
- Группирует screenings по стадиям
- Управляет видимостью rejected-колонки
- Открывает модальное окно при клике на карточку

#### `KanbanColumn.jsx`
Колонка с кандидатами
- Заголовок с названием стадии и счетчиком
- Скроллируемый список карточек
- Empty State если нет кандидатов

#### `KanbanCard.jsx`
Компактная карточка кандидата
- Аватар, имя, время
- Score badge с цветовой индикацией (зеленый >80, желтый 60-80, серый <60)
- Иконки резюме и Telegram
- Hover эффект с подъемом карточки
- Бейдж "Попытка N" для повторных

#### `CandidateModal.jsx`
Модальное окно с детальной информацией
- Overlay с затемнением (rgba(0,0,0,0.5))
- Кнопка закрытия (X)
- Переиспользует `CandidateCard.jsx` для контента
- Анимация появления (fade-in + scale-up)
- Закрытие по Escape, клик вне окна, или кнопка X

### Утилиты:

#### `utils/pipelineStages.js`
- `PIPELINE_STAGES` - конфигурация всех стадий
- `getScreeningStage(screening)` - определяет стадию кандидата
- `groupScreeningsByStage(screenings, showRejected)` - группирует по стадиям
- `getStageConfig(stageKey)` - получает конфигурацию стадии

## 🔧 Интеграция и использование

### В `Candidates.jsx`:

```jsx
import KanbanBoard from '../components/KanbanBoard'

// Состояния:
const [showRejected, setShowRejected] = useState(false)
const [showRepeated, setShowRepeated] = useState(true)

// Фильтрация повторных:
const filtered = showRepeated
  ? screenings
  : screenings.filter((s) => (s.screening_index ?? 1) === 1)

// Рендер:
<KanbanBoard
  screenings={filtered}
  vacancyTitle={selectedVacancy?.title}
  showRejected={showRejected}
  onStatusChange={handleStatusChange}
/>
```

## 🐛 Известные ограничения

1. **Группировка на клиенте**
   - Все screenings загружаются сразу, группировка происходит на фронте
   - При большом количестве кандидатов (>100) может тормозить
   - Решение: API endpoint для группированных данных

2. **Нет drag-and-drop**
   - Пока можно изменить статус только через модалку
   - Планируется в Phase 2

## 📝 Примеры использования

### Как открыть страницу кандидатов для конкретной вакансии:

```
http://localhost:5173/candidates?vacancy_id=<uuid>
```

### Как показать отклоненных кандидатов:

Включить чекбокс "Показать отклоненных" над Kanban-доской

### Как скрыть повторные отклики:

Выключить чекбокс "Повторные" над Kanban-доской
