# Обновление доступа к вакансиям

## Дата: 2026-05-26

### Изменения в RBAC для вакансий

#### 1. Все сотрудники компании видят вакансии компании

**Было**: Каждый сотрудник видел только свои собственные вакансии (фильтр по `user_id`).

**Стало**: Все сотрудники агентства видят вакансии всех коллег из своего агентства (фильтр по `agency_id`).

**Файлы**: `app/routers/vacancies.py`

**Измененные эндпоинты**:
- `GET /vacancies/` - показывает все вакансии агентства
- `GET /vacancies/{vacancy_id}` - позволяет просматривать любую вакансию агентства
- `PATCH /vacancies/{vacancy_id}/status` - позволяет обновлять статус любой вакансии агентства
- `PATCH /vacancies/{vacancy_id}` - позволяет редактировать любую вакансию агентства
- `PATCH /vacancies/{vacancy_id}/archive` - позволяет архивировать любую вакансию агентства
- `GET /vacancies/{vacancy_id}/stats` - показывает статистику по любой вакансии агентства

#### 2. Recruiter может удалять вакансии

**Было**: Удалять вакансии мог только администратор (`@require_role(UserRole.ADMIN)`).

**Стало**: Удалять вакансии агентства может любой сотрудник (admin и recruiter).

**Файлы**: `app/routers/vacancies.py`

**Измененный эндпоинт**:
- `DELETE /vacancies/{vacancy_id}` - убран декоратор `@require_role(UserRole.ADMIN)`

### Тестирование

Создан и успешно выполнен тест `test_team_vacancies.py`, который проверяет:
1. Администратор создает вакансию
2. Администратор приглашает recruiter'а
3. Recruiter принимает приглашение и логинится
4. Recruiter видит вакансии администратора в списке
5. Recruiter может успешно удалить вакансию администратора

**Результат**: Все тесты пройдены успешно.

### Рекомендации

- **Права доступа**: Все сотрудники агентства имеют полный доступ ко всем вакансиям. Если в будущем потребуется более детальная система прав (например, recruiter может только просматривать вакансии других), нужно будет доработать RBAC.
- **Аудит**: Рекомендуется добавить логирование операций удаления вакансий для отслеживания действий сотрудников.
