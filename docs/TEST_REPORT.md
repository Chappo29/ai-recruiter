## 🚨 НАЙДЕННЫЕ БАГИ И ИСПРАВЛЕНИЯ

### БАГ #1: Невозможно добавить второго пользователя в компанию
**Проблема:** Каждая регистрация создавала новое агентство. Не было способа пригласить сотрудников.

**✅ ИСПРАВЛЕНО:**
- Создан новый роутер `/team` с endpoint'ами:
  - `POST /team/invite` - admin приглашает пользователя (возвращает токен)
  - `POST /team/accept-invite` - пользователь принимает приглашение
  - `GET /team/invitations` - список активных приглашений
- Токены приглашений действуют 24 часа
- Приглашенные пользователи автоматически получают указанную роль

**Новые файлы:**
- `app/routers/team.py` - роутер управления командой
- `app/schemas/team.py` - схемы для приглашений

## 📝 ДОПОЛНИТЕЛЬНЫЕ НАХОДКИ

### ⚠️ Потенциальные улучшения:

1. **Токены приглашений в памяти**
   - Текущая реализация: in-memory storage (dict)
   - Проблема: при перезапуске API токены теряются
   - Рекомендация: использовать Redis или БД

2. **Нет endpoint'а для удаления пользователей**
   - Admin не может удалить пользователя из команды
   - Рекомендация: добавить `DELETE /team/users/{user_id}`

3. **Нет endpoint'а для изменения роли**
   - Admin не может повысить recruiter'а до admin
   - Рекомендация: добавить `PATCH /team/users/{user_id}/role`

4. **Список пользователей агентства**
   - Нет endpoint'а для просмотра всех пользователей компании
   - Рекомендация: добавить `GET /team/users`

5. **Вакансии видны только создателю**
   - В `list_vacancies` фильтр: `Vacancy.user_id == current_user.id`
   - Проблема: recruiter не видит вакансии admin'а своего агентства
   - Рекомендация: фильтровать по `agency_id` вместо `user_id`
